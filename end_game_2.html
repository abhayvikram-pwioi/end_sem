JavaScript Web Developer (Advanced JS) - Question Paper Set 1

Total Marks: 60 | Time: 3 Hours

Part A (30 Marks)

Answer any 5 questions. Each question carries 5 marks.

Q1. Create a simple HTML page with a form containing two number inputs and a button labeled "Calculate Sum". On button click, use JavaScript to display the sum in a green highlighted paragraph below the form. Use let and const appropriately. Validate that both inputs are numbers using conditionals. Integrate basic CSS for styling the form and result area.

[5 Marks]

Q2. Build an HTML page with a button "Toggle Theme". Clicking it should switch the background color of the body between dark (#121212) and light (#ffffff) mode, and update text color accordingly. Use a function declaration to handle the toggle logic with block scope variables. Add smooth transition using CSS.

[5 Marks]

Q3. Create a dynamic list using an array of objects (e.g., students with name and score). Render the list as an unordered HTML list. Include a search input that filters the list in real-time using array methods like filter() and DOM updates. Style the list with alternating row colors using CSS.

[5 Marks]

Q4. Create a webpage with a product card (image, name, price) fetch from http://dummyjson.com/products. Use a Set to track "wishlist" items. Add "Add to Wishlist" button that prevents duplicates and shows total wishlist products. Persist wishlist using localStorage.

[5 Marks]

Q5. Create two sections: one for adding tasks (array), another for completed tasks. Move tasks between them on button click.

[5 Marks]

Part B (30 Marks)

Answer all 2 questions. Each carries 10 marks.

Q6. Develop a "Todo Application" UI with HTML form (input + add button), a list of todos, and filters (All, Active, Completed). Implement add, delete, and toggle complete functionality. Handle errors if input is empty. Style with responsive CSS Flexbox/Grid and hover effects.

[10 Marks]

Q7. Build a "User Profile Manager" page. Allow creating/delete user objects (name, email, skills as Set). Use OOP-style constructor or class-like structure with methods. Implement form validation with try-catch for error handling. Display list of users with search. Persist using localStorage.

[10 Marks]
