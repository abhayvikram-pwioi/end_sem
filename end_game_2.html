JavaScript Web Developer (Advanced JS) - Question Paper Set 1

Total Marks: 60 | Time: 3 Hours

Part A (30 Marks)

Answer any 6 questions. Each question carries 5 marks.

Q1. Create a simple HTML page with a form containing two number inputs and a button labeled "Calculate Sum". On button click, use JavaScript to display the sum in a green highlighted paragraph below the form. Use let and const appropriately. Validate that both inputs are numbers using conditionals. Integrate basic CSS for styling the form and result area. (Focus: Variables, Conditionals, DOM manipulation)

[5 Marks]

Q2. Build an HTML page with a button "Toggle Theme". Clicking it should switch the background color of the body between dark (#121212) and light (#ffffff) mode, and update text color accordingly. Use a function declaration to handle the toggle logic with block scope variables. Add smooth transition using CSS. (Focus: Functions, Scope, Event Handling, CSS integration)

[5 Marks]

Q3. Create a dynamic list using an array of objects (e.g., students with name and score). Render the list as an unordered HTML list. Include a search input that filters the list in real-time using array methods like filter() and DOM updates. Style the list with alternating row colors using CSS. (Focus: Arrays, Array methods, DOM, Event listeners)

[5 Marks]

Q4. Design an HTML calculator interface with buttons for digits 0-9, +, -, *, /, and =. Implement the core logic using a Map to store operator functions. Handle division by zero gracefully with custom error display. Use CSS Grid for button layout. (Focus: Functions as values, Map, Error handling, UI layout)

[5 Marks]

Q5. Create a webpage with a product card (image, name, price). Use a Set to track "wishlist" items. Add "Add to Wishlist" button that prevents duplicates and shows a count badge. Persist wishlist using localStorage. Style with modern card CSS. (Focus: Set, localStorage, Objects, DOM)

[5 Marks]

Q6. Build a simple module-like structure using IIFE or separate script concepts (in single HTML). Create two sections: one for adding tasks (array), another for completed tasks. Move tasks between them on button click. Demonstrate block scope and hoisting awareness by proper variable placement. (Focus: Scopes, Hoisting, Arrays, Modular thinking)

[5 Marks]

Part B (30 Marks)

Answer all 3 questions. Each carries 10 marks.

Q7. Develop a "Todo Application" UI with HTML form (input + add button), a list of todos, and filters (All, Active, Completed). Implement add, delete, and toggle complete functionality. Use ES6 modules syntax simulation (comments), async/await simulation with Promises for "saving" to localStorage. Handle errors if input is empty. Style with responsive CSS Flexbox/Grid and hover effects. (Focus: Full CRUD with Arrays/Objects, Event delegation, localStorage, Async concepts, UI handling)

[10 Marks]

Q8. Create a Quiz interface with multiple questions stored in an array of objects. Display one question at a time with radio buttons for options. Implement next/previous navigation, timer (using setInterval), and score calculation at the end. Use Map for question-answer mapping. Show results with percentage and confetti-like CSS animation on high score. Handle edge cases like time expiry. (Focus: Arrays, Conditionals, Timers, DOM, Complex state management)

[10 Marks]

Q9. Build a "User Profile Manager" page. Allow creating/editing user objects (name, email, skills as Set). Use OOP-style constructor or class-like structure with methods. Implement form validation with try-catch for error handling. Display list of users with search. Persist using localStorage. Include dark mode toggle. Demonstrate understanding of prototypes/objects. (Focus: OOP concepts, Error Handling, Set/Map, Forms, Advanced UI + State)

[10 Marks]
